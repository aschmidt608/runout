Load the collection
Discogs doesn't allow browser-based requests with a personal token, so pull your collection with a one-time local script instead; your token never leaves your machine.
  1. Get a token at Discogs → Settings → Developers → "Generate new token."
  2. Download fetch_discogs_collection.py (shared alongside this app) and run:
    python fetch_discogs_collection.py YOUR_USERNAME YOUR_TOKEN
    Optionally add a free GetSongBPM API key as a third argument to also look up BPM per release (coverage will be partial).
  3. Drop the discogs_collection.json it writes into the box below.

Assumptions baked in: your Claude API key, Tidal session, and collection all live only in this browser's local storage — nothing is sent anywhere except directly from your device to Anthropic's and Tidal's own APIs, and nothing persists if you clear your browser data or switch devices. The collection comes from a local JSON file (Discogs' own CSV export leaves out genre/style, and browser-based requests with a personal token aren't practical anyway), so re-run the script when your collection changes. Track duration isn't pulled per-release either, so length is estimated from the avg-minutes-per-track field above. BPM comes from an optional, best-effort GetSongBPM lookup in the fetch script — coverage is partial (weakest for obscure or vinyl-only pressings), so the BPM range filter only excludes releases with a known BPM outside it; anything without a match passes through and falls back to the tempo/mood hint instead. Pushing to Tidal matches each track by artist/title search and takes the first hit, so double-check the result — obscure or rare pressings may not match anything on Tidal at all, and your Tidal login expires after a few hours (no refresh yet), so you'll occasionally need to reconnect.
